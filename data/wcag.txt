한국형 웹 콘텐츠 접근성 지침 2.2 요약
이 지침은 장애인과 고령자를 포함한 모든 사용자가 웹 콘텐츠에 동등하게 접근할 수 있도록 하는 기준을 제공합니다. 주요 원칙과 지침은 다음과 같습니다:

원칙 1: 인식의 용이성 (Perceivable)
사용자가 콘텐츠를 인식할 수 있도록 정보와 사용자 인터페이스 구성 요소를 제공해야 합니다.

지침 1.1 대체 텍스트

1.1.1 적절한 대체 텍스트 제공: 텍스트 아닌 콘텐츠는 의미나 용도를 이해할 수 있는 대체 텍스트를 제공해야 합니다.
지침 1.2 멀티미디어 대체 수단

1.2.1 자막 제공: 멀티미디어 콘텐츠에는 자막, 대본 또는 수어를 제공해야 합니다.
지침 1.3 적응성

1.3.1 표의 구성: 표는 이해하기 쉽게 구성해야 합니다.
1.3.2 콘텐츠의 선형 구조: 콘텐츠는 논리적인 순서로 제공되어야 합니다.
1.3.3 명확한 지시사항 제공: 지시사항은 모양, 크기, 위치, 색 등에 관계없이 인식될 수 있어야 합니다.
지침 1.4 명료성

1.4.1 색에 무관한 콘텐츠 인식: 콘텐츠는 색에 관계없이 인식될 수 있어야 합니다.
1.4.2 자동 재생 금지: 자동으로 소리가 재생되지 않아야 합니다.
1.4.3 텍스트 콘텐츠의 명도 대비: 텍스트와 배경 간의 명도 대비는 4.5:1 이상이어야 합니다.
1.4.4 콘텐츠 간의 구분: 이웃한 콘텐츠는 구별될 수 있어야 합니다.
원칙 2: 운용의 용이성 (Operable)
사용자가 사용자 인터페이스 구성 요소와 내비게이션을 조작할 수 있도록 해야 합니다.

지침 2.1 입력장치 접근성

2.1.1 키보드 사용 보장: 모든 기능은 키보드만으로도 사용할 수 있어야 합니다.
2.1.2 초점 이동과 표시: 키보드에 의한 초점은 논리적으로 이동해야 하며, 시각적으로 구별할 수 있어야 합니다.
2.1.3 조작 가능: 사용자 입력 및 콘트롤은 조작 가능하도록 제공되어야 합니다.
2.1.4 문자 단축키: 문자 단축키는 오동작으로 인한 오류를 방지해야 합니다.
지침 2.2 충분한 시간 제공

2.2.1 응답시간 조절: 시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 합니다.
2.2.2 정지 기능 제공: 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 합니다.
지침 2.3 광과민성 발작 예방

2.3.1 깜빡임과 번쩍임 사용 제한: 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 합니다.
지침 2.4 쉬운 내비게이션

2.4.1 반복 영역 건너뛰기: 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 합니다.
2.4.2 제목 제공: 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 합니다.
2.4.3 적절한 링크 텍스트: 링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 합니다.
2.4.4 고정된 참조 위치 정보: 전자출판문서 형식의 웹 페이지는 각 페이지로 이동할 수 있는 기능이 있어야 합니다.
지침 2.5 입력 방식

2.5.1 단일 포인터 입력 지원: 다중 포인터 또는 경로 기반 동작은 단일 포인터 입력으로도 조작할 수 있어야 합니다.
2.5.2 포인터 입력 취소: 단일 포인터 입력으로 실행되는 기능은 취소할 수 있어야 합니다.
2.5.3 레이블과 네임: 사용자 인터페이스 구성 요소는 네임에 시각적으로 표시되는 텍스트를 포함해야 합니다.
2.5.4 동작 기반 작동: 동작 기반으로 작동하는 기능은 사용자 인터페이스로 조작할 수 있고, 해당 기능을 비활성화할 수 있어야 합니다.
원칙 3: 이해의 용이성 (Understandable)
사용자가 콘텐츠를 이해하고 조작할 수 있도록 해야 합니다.

지침 3.1 가독성

3.1.1 기본 언어 표시: 주로 사용하는 언어를 명시해야 합니다.
지침 3.2 예측 가능성

3.2.1 사용자 요구에 따른 실행: 사용자가 의도하지 않은 기능은 실행되지 않아야 합니다.
3.2.2 찾기 쉬운 도움 정보: 도움 정보는 각 페이지에서 동일한 상대적인 순서로 접근할 수 있어야 합니다.
지침 3.3 입력 도움

3.3.1 오류 정정: 입력 오류를 정정할 수 있는 방법을 제공해야 합니다.
3.3.2 레이블 제공: 사용자 입력에는 대응하는 레이블을 제공해야 합니다.
3.3.3 접근 가능한 인증: 인증 과정은 인지 기능 테스트에만 의존해서는 안 됩니다.
3.3.4 반복 입력 정보: 반복되는 입력 정보는 자동 입력 또는 선택 입력할 수 있어야 합니다.
원칙 4: 견고성 (Robust)
콘텐츠가 다양한 사용자 에이전트와 보조기술과 호환되어야 합니다.

지침 4.1 문법 준수

4.1.1 마크업 오류 방지: 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 합니다.
지침 4.2 웹 애플리케이션 접근성

4.2.1 웹 애플리케이션 접근성 준수: 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 합니다.
